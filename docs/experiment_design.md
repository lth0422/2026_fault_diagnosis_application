# 실험 설계서 (저널화 로드맵)

문헌 조사(19편 정독)를 바탕으로, 현재 앱/데이터를 **저널 논문**으로 끌어올리기 위한 실험을
구체적으로 설계한다. 세 축을 검증한다: **① 신뢰성(누수 없는 평가) ② 강건성(다조건) ③ novelty(스마트폰 실용성)**.

> 원칙(사용자 피드백 반영): 목표는 앱 확장 + 논문화. "최고 정확도"가 아니라 **접근성·강건성·정직성**이 핵심 가치.
> 관련: [data_leakage_analysis.md] [fps_temporal_resolution_analysis.md] [paper_concept.md] [related_works.md]

---

## 0. 먼저 확정해야 할 사실 (실험 전 확인)

| 항목 | 왜 필요 | 확인 방법 |
|------|---------|-----------|
| 실제 데이터 분할 방식 | 누수 재평가의 baseline | 코드/기록에서 10-fold가 random인지, 녹화/베어링 단위 분리 있는지 |
| 클래스당 영상/녹화 세션 수 | recording-wise 분할 가능 여부 | 원본 영상 파일 목록 |
| 실제 베어링 제원(n,d,D,θ) | BPFO/BPFI/BSF 계산 | 베어링 모델명/스펙시트 |
| 마커→변위 매핑 레벨 | 불확도 예산(C4) | 앱 코드: scale factor(Level0)인지 |
| 촬영 거리·조명·기기 | 강건성 baseline 기록 | 실험 로그 |

## 실험 1 — 데이터 누수 재평가 (⭐ 최우선, 신뢰성 축)

**목적:** 99.7%가 누수로 부풀려졌는지 정량화하고, 정직한 성능 범위를 확보.

### 1-A. 분할 방식 비교 (누수 곡선)
근거: E1(part-to-part로 오차 47%p↑), E2(segmentation/bearing-wise), WDCNN원논문(train만 overlap).

| 조건 | 분할 방법 | 예상 |
|------|-----------|------|
| S0 (현행 재현) | 전체 중첩증강 후 random 10-fold | ~99.7% |
| S1 | 중첩 제거 + 시간순 blocked (앞70%/뒤30%) | 하락 |
| S2 | recording-wise (영상 단위 분리) | 더 하락 |
| S3 | (가능시) bearing-wise | 최저=진짜 일반화 |

- **산출물:** S0→S3 정확도 곡선. 하락 폭이 누수 크기. **이 곡선 자체가 논문 기여.**

### 1-B. 평가지표 교체
- accuracy → **balanced accuracy + Macro AUROC**(E2, prevalence-independent).
- multi-label 이진공식화(IR/OR/B 독립 detector, 정상=전부0) 검토(E2/E5).

### 1-C. 혼동행렬 + logits
- IR→OR 오분류(Android/iOS) 정량화. 경계사례 vs 누수붕괴 구분.

## 실험 2 — fps / 시간해상도 유효범위 (novelty+한계 축)

**목적:** "스마트폰 240fps로 무엇이 잡히고 안 잡히는가"를 물리적으로 규명.
근거: B3(5000fps), M3(1000fps지만 저주파만), C4(저샘플링 미분 노이즈), fps 분석문서.

### 2-A. 결함주파수 vs Nyquist 경계표
- 실제 베어링 제원으로 RPM별 BPFO/BPFI/BSF 계산 → 240fps Nyquist(120Hz)와 대조.
- **적용 가능 RPM 상한**을 명시(예: BPFO<120Hz 되는 속도).

### 2-B. DisplacementZ 주파수 분석
- 우리 신호의 FFT에서 **어떤 성분이 클래스 구분에 기여**하는지(저차 BPFO 존재?).
- 저주파에 판별정보 없으면 → 실험1의 누수 결론과 연결.

### 2-C. (가능시) 240 vs 960fps 비교
- 최신폰 960fps 모드로 재촬영 → fps↑시 성능 변화. (Yu 2021 future work 실증)

## 실험 3 — 다조건 강건성 (강건성 축)

**목적:** 단일 조건(1200rpm/연속광/삼각대/iPhone) 한계 극복.
근거: C2(각도×스케일×조명→상관), 조명분해(Xiong), 자이로보정(Yu), C4(불확도예산).

### 통제변수 실험 (한 번에 한 변수)
| 변수 | 범위 | 측정 |
|------|------|------|
| RPM | 600·900·1200·1500(가능범위) | 정확도 + BPFO 위치 |
| 조명 | 연속광·형광등·저조도·역광 | 마커검출률 + 정확도 |
| 거리·각도 | 근/원, 정면/사선 | 검출률 + 정확도 |
| 기기 | iPhone·Android(다른 fps) | 정확도 |
| 흔들림 | 삼각대 vs 핸드헬드 | zStdDev + 정확도 |

- **평가:** C2식 상관계수 + 정확도. **C4 불확도 예산 템플릿으로 측정품질 보고.**
- 저조도 대응: Xiong 조명-반사분해 / 각도 대응: Yu 자이로보정 검토.

---

## 실험 4 — 소규모 데이터 / 일반화 (모델 축, 선택)

**목적:** 1060샘플 소규모 + 단일베어링 한계 완화.
근거: D6(CWRU 사전학습→전이, Siamese-WDCNN, cold-start 85~89%).

- **4-A.** CWRU로 WDCNN 사전학습 → 우리 데이터 few-shot 전이학습.
- **4-B.** Siamese-WDCNN(대조학습)으로 소규모 대응.
- **4-C.** (근본) 클래스당 복수 물리 베어링 추가 촬영 → bearing-wise 평가 가능화.

## 실험 5 — 앱 확장 (플랫폼 축, 코드작업/승인 필요)

**목적:** 앱을 논문 실험 도구이자 실용성 데모로.

| 확장 | 목적 | 난이도 |
|------|------|--------|
| 다조건 메타데이터 기록(RPM/거리/각도/조명/기기) | 실험3 데이터 자동 태깅 | 낮음 |
| CSV/logits 내보내기 강화 | 실험1 재현 | 낮음 |
| 자이로 off-axis 보정 | 실험3 각도강건성 | 중간 |
| 마커리스 모드(phase/optical flow) | 옵션A, 마커의존 제거 | 높음 |
| 신뢰도 표시(softmax+검출품질) | 현장 신뢰성 | 낮음 |

> 코드 수정은 지금까지 "손대지 말라" 지침 → 착수 전 승인 필요.

## 우선순위 & 논문 스토리라인

**우선순위:** 실험0(사실확인) → 실험1(누수, 최우선) → 실험2(fps) → 실험3(강건성) → 실험4/5(선택).

**논문 한 줄:**
> "별도 장비 없이 스마트폰으로 현장 End-to-End 베어링 진단이 가능한가? 우리는 저비용·저fps 영역에서
> 이를 구현하고, **누수 없는 정직한 평가**로 실제 성능과 물리적 적용범위(저·중속)를 처음으로 규명한다."

**기여 3점:**
1. 스마트폰 온디바이스 베어링 진단 시스템(앱) — 접근성.
2. 다조건 강건성 + 측정 불확도 정량 보고 — 실용성/신뢰성.
3. 카메라 기반 진단에 누수 없는 평가 프로토콜 도입 — 방법론.
