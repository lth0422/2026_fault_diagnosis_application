# 5. 그 외 조사 정리

앞 4개 페이지에 담기지 않은 조사 결과와 실무 이슈를 모음.

## 5-1. 문헌 조사 현황
- 관련 논문 **54편 수집**(블록 A~F + Motion Magnification), **19편 정독**(2026-07-23 기준).
- 블록: A 마커리스 변위추출 / B 카메라 기반 진단 / C 촬영환경 강건성 / D 도메인적응·WDCNN / E 데이터누수 / F 스마트폰 계측 / M 모션확대.
- 정독 목록: E1·E2(누수), TICNN, Peng·WDCNN원논문, B1 이벤트카메라·B3 5000fps·RESS, C2·조명분해·C4 불확도, D6 few-shot, F 240fps·SURVISHNO·자이로·저비용SHM, Wadhwa·M3.

## 5-2. 모델 계보 정정 (중요)
- 우리 모델(`Fwdcnn7.ptl`) = **WDCNN**. 원논문은 **Zhang, Sensors 2017**(우리 참고문헌 [2]).
- 라이너가 "WDCNN 원논문"이라 표기한 **MSSP 2018(1033회 인용)은 사실 후속작 TICNN**.
- WDCNN 구조: 입력 2048×1 → **첫 층 wide 커널 64×1**(고주파 잡음 억제) → 3×1 small 커널 5층 → FC100 → softmax. 우리 클래스 순서 B/H/IR/OR.
- **주의:** 원논문 PDF가 `papers/Block A/` 폴더에 잘못 분류돼 있음.

## 5-3. 소규모 데이터 해법
- 우리 데이터는 1,060샘플로 WDCNN 기준 소규모(WDCNN은 대량 필요: 90샘플 82% → 19,800샘플 100%).
- **해법(D6, Sci.Reports 2025):** CWRU 공개 데이터로 WDCNN 사전학습 → 우리 데이터로 few-shot 전이학습.
- cold-start 정확도: WDCNN 70% → FSL+TL 85% → +Attention 88.75% (정직하게 평가한 현실적 수치).
- 관찰: **"저속일수록 결함 특징이 증폭"** → 우리 저속 영역과 일치(긍정 신호).

## 5-5. 앱(플랫폼) 관련 실무 이슈
- 현재 앱: Flutter(Android/iOS), OpenCV 마커추적 + PyTorch/LibTorch 온디바이스 추론까지 구현됨. **논문 실험 도구로 바로 활용 가능.**
- **Android/iOS 일관성 이슈**: iPhone에서 IR 영상 1개가 OR로 오분류. Android/iOS의 OpenCV 변위 계산 미세 차이 가능 → 동일 영상 CSV·logits 비교 필요(모델 경계사례 vs 구현차이 구분).
- 앱 확장 후보: 다조건 메타데이터 자동 기록, CSV/logits 내보내기 강화, 자이로 off-axis 보정, 신뢰도 표시, (장기) 마커리스 모드.

## 5-6. 측정 신뢰성 프레임 (C4)
- 영상 변위법은 "알고리즘"이 아니라 **측정 시스템**으로 다뤄야: 측정모델 + 불확도 예산 + 타당성 조건 + 참조 비교.
- 우리 파이프라인은 **Level 0(scale factor)** 매핑으로 추정 → 타당성 조건(소ROI·직각촬영·out-of-plane 없음) 명시 필요.
- 논문 methods에 GUM/Monte Carlo 기반 불확도 예산을 넣으면 신뢰성이 크게 올라감.
