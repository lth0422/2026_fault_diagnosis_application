# 3. 촬영 환경 적합성 · 강건성 조사

## 결론
현재 실험은 **단일 조건**(1200 RPM · 연속광 · 삼각대 · iPhone 12 Pro · 고정 거리)이라 강건성이 검증되지 않았다. 특히 **스마트폰 240fps는 고주파 베어링 결함주파수를 물리적으로 못 잡는다**(저·중속 한정). 이는 한계이자 동시에 차별점이며, 체계적 강건성 실험이 필요하다.

## 3-1. fps / 시간해상도 — 가장 근본적 이슈
### 핵심: 240fps → Nyquist 120Hz
| 연구 | fps | Nyquist | 비고 |
|------|----:|--------:|------|
| **우리** | **240** | **120 Hz** | 저주파 변위만 |
| Peng [1] | 1000 | 500 Hz | 베어링 4클래스 |
| **Zhou [2] (최유사)** | **5000** | **2500 Hz** | BPFO~126/BPFI~204Hz@1800rpm 포착 |
| 이벤트카메라 [3] | μs급 | ≫kHz | 특수 하드웨어 |
| 가속도계(참조) | 8192~20000 | 4~10kHz | — |

- 베어링 결함주파수(1200rpm 기준): **BPFO ~84Hz, BPFI ~136Hz**. → BPFI는 이미 120Hz 초과, 고조파는 전부 손실 [2].
- **속도를 올리면(≥1500rpm) BPFO조차 못 잡음.** → 우리가 **1200rpm 단일조건을 쓴 것이 사실상 fps 천장에 걸린 결과**일 수 있음.
- 영상법 자체가 저주파 편향: EVMM 연구 [4]도 1000fps인데 1×/2×(24~48Hz)만 분석, 비정상조건 진단 [5]도 영상은 회전/order 추적만 하고 결함주파수는 가속도계로 검증.

### 단, 저주파 계측은 타당
- 스마트폰 240fps 검증 [6]: **우리와 거의 동일 셋업**(Redmi Note 8, 240fps, 대비색 마커, 삼각대 150cm)으로 기준장비(LPT) 대비 **ICC 0.99+**. → "240fps 마커추적은 저주파 변위엔 신뢰 가능". 문제는 대역이지 계측 정확도가 아님.
- 단, 영상은 샘플링이 낮아 변위를 미분해 가속도를 얻으면 노이즈가 증폭되므로 변위 영역에 머무는 것이 타당 [10].

## 3-2. 촬영 환경 변수 (미검증)
| 변수 | 현재 | 미검증 |
|------|------|--------|
| RPM | 1200 고정 | 가변속도, 기동/정지 |
| 조명 | 연속광 | 형광등·자연광·저조도·역광 |
| 카메라 | 삼각대 고정 | 핸드헬드 흔들림, 각도 변화 |
| 기기 | iPhone 12 Pro | 다른 스마트폰·해상도·fps |
| 거리 | 고정 | 거리 변화 |
| 배경 | 실험실 | 산업 현장(기름·먼지) |

## 3-3. 강건성 평가 방법 (문헌 템플릿 확보)
- **촬영각도×스케일×조명 프로토콜** [7]: 6조건을 **상관계수**로 평가. 결론: **①조명이 가장 치명적 ②근접/줌(local)≫원거리(global) ③진동방향과 직각 촬영 필수**.
- **조명 보정** [8]: 조명-반사 분해로 왜곡 보정(MAC 0.75→0.95).
- **각도 보정** [9]: 스마트폰 **내장 자이로**로 off-axis 촬영 보정(각도오차<0.7°).
- **불확도 보고** [10]: 영상변위법을 측정시스템으로 다뤄 불확도 예산 작성(우리 파이프라인은 Level 0 scale-factor 매핑, 타당성 조건 명시 필요).

## 강건성 실험 설계 (제안)
한 번에 한 변수씩 통제: RPM·조명·거리/각도·기기·흔들림 → **상관계수 + 정확도**로 측정 [7]. 저조도는 조명분해 [8], 각도는 자이로보정 [9] 검토.

## 회의 논의점
- [ ] RPM 적용 범위를 **fps 한계로 명시**할 것인가(예: ≤1200rpm)? 아니면 960fps 모드로 확장 실험 [9]?
- [ ] 강건성 실험 우선순위(조명 > 각도/거리 > 기기 > 흔들림 추천) [7].
- [ ] 실제 베어링 제원 확보 → BPFO/BPFI 경계표 작성 필요 [2].
